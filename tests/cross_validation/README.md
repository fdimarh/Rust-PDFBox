# Cross-Validation Reference Snapshots

This directory contains reference JSON snapshots generated by the canonical
Java PDFBox implementation. The Rust port is validated against these snapshots
in `tests/cross_validate.rs`.

## Format

Each `.json` file corresponds to one PDF fixture. Fields:

```json
{
  "file": "smoke/letter_single_page.pdf",
  "pdf_version": "1.4",
  "page_count": 1,
  "pages": [
    {
      "index": 0,
      "width": 612.0,
      "height": 792.0,
      "rotation": 0,
      "text_len_min": 0,
      "text_len_max": 9999,
      "text_contains": []
    }
  ],
  "permissions": {
    "print": true,
    "copy": true,
    "modify": true,
    "annotate": true
  },
  "fonts": [],
  "metadata": {
    "title": null,
    "author": null
  }
}
```

## Tolerances

- Page dimensions: ±0.5 pt  
- Text length: within declared `[min, max]` range  
- Text contains: all strings in `text_contains[]` must appear  
- Permissions: exact match  

## Regenerating

To regenerate from a live Java PDFBox install:

```bash
# Requires Java + pdfbox-app-*.jar on PATH
java -jar pdfbox-app.jar ExtractText -i tests/fixtures/smoke/letter_single_page.pdf
```

The JSON snapshots were captured once and committed for deterministic CI.
